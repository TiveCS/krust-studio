# Krust Studio v1.2.2

A big feature release for the schema-editing workflow, plus auto-update and multi-database support. (v1.2.2 = v1.2.0 with the Windows installer build + packaging fixed — no app-behavior changes.)

> ⚠️ Personal project, vibe-coded and open-sourced. **Use at your own risk** — always have backups before editing real data.

## ✨ Highlights

- **Staged, reviewable schema edits.** Column edits *and* index add/drop now stage together, are colour-coded (🟢 new · 🟡 changed · 🔴 dropped · 🔵 moved), and commit in **one batch**. Committing opens a panel showing the **exact DDL** before anything runs.
- **Column reorder** by drag (MySQL/MariaDB), with new columns keeping their position (`AFTER`).
- **Auto-update** — the app checks GitHub Releases on launch, downloads in the background, and prompts **Restart now**.
- **Optional database / multi-database switcher** — leave the database blank to connect at the server level and switch databases from the sidebar.
- **Ctrl/⌘+P command palette** to jump to any table.

## 🚀 Added

- Auto-update via GitHub Releases; installer lets you **choose the install folder** (Windows).
- Optional database name + database switcher (MySQL `USE` in place; PostgreSQL reconnects).
- Ctrl/⌘+P table search palette.
- SQL editor: schema + **column** autocomplete (lazy, alias-aware), **engine-aware** dialect/quoting (backtick vs double-quote), resizable editor/results split, cancel.
- DDL viewer is now syntax-highlighted; structure commits show a DDL review panel.
- Column reorder (drag) and **staged index add/drop**, committed together.
- Engine-aware index method picker (HASH disabled on InnoDB/MyISAM, where it's silently a B-tree).

## 🐛 Fixed

- Faithful MySQL `MODIFY` — type/null/default edits no longer **silently drop** `AUTO_INCREMENT`, default, or collation.
- Tab bar stays pinned on long tables; query results fill the pane.
- Column-resize, the editor/results split, and SQL typing no longer re-render the whole grid per interaction.
- Sidebar connection/database switcher no longer lags on large schemas.
- Connection editor no longer stays stuck open when you click a table.
- JSON viewer renders datetimes correctly (was `{0}`).
- Auto-LIMIT now records the **actual** executed SQL (`… LIMIT 500`).
- **Windows installer build** fixed (directory picker requires the assisted installer).

## 📦 Install

| OS | File |
|----|------|
| **Windows** | `krust-studio-app-1.2.2-setup.exe` |

Not code-signed, so Windows SmartScreen may warn the first time → **More info → Run anyway**. The installer lets you pick the install location. Future versions auto-update from here.

## ⚠️ Notes

- **MySQL DDL auto-commits per statement** — a multi-statement schema commit isn't atomic (a mid-batch failure can leave earlier statements applied). Review the DDL before committing. True rollback only on PostgreSQL.
- Column **reorder** and new-column **positioning** are MySQL-only (Postgres/SQLite can't reorder without a table rebuild, which Krust refuses by design).
